# Git Workflow Guide for Triadic

## Initial Setup (One-time)

### 1. Configure Git (if not already done)
```bash
git config --global user.name "Your Name"
git config --global user.email "your.email@example.com"
```

### 2. Create GitHub Repository
- Go to https://github.com/new
- Create a new repository (e.g., `triadic`)
- Don't initialize with README

### 3. Connect Local Repository to GitHub
```bash
git remote add origin https://github.com/YOUR_USERNAME/triadic.git
git branch -M main
```

## Daily Workflow

### Making Changes and Pushing Updates

1. **Make your code changes** (edit files, etc.)

2. **Check what changed:**
   ```bash
   git status
   ```

3. **Stage your changes:**
   ```bash
   # Stage all changes
   git add .
   
   # Or stage specific files
   git add app.py utils/streamlit_ui.py
   ```

4. **Commit your changes:**
   ```bash
   git commit -m "Description of what you changed"
   ```

5. **Push to GitHub (when ready):**
   ```bash
   git push origin main
   ```

### Useful Commands

**View changes before committing:**
```bash
git diff
```

**View commit history:**
```bash
git log --oneline
```

**Undo changes (before committing):**
```bash
# Discard changes to a file
git checkout -- filename.py

# Discard all uncommitted changes
git reset --hard HEAD
```

**Update from remote (if working on multiple machines):**
```bash
git pull origin main
```

## Streamlit Cloud Deployment

Once you push to GitHub, Streamlit Cloud will automatically:
- Detect the push
- Redeploy your app with the latest changes

You can also manually trigger a redeploy from the Streamlit Cloud dashboard.
